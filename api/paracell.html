load_channels(reload=False) → None[source]¶

Loads channels into memory as numpy arrays.

For instance, if you’ve instantiated an FOV object with a prefix D6–W00024–P00003–Z00002–T00000 and you have the following channels: DAPI, FITC, Cyan, Cy3, Cy5 It can load all of these channels into memory.

Parameters:

  • parent_path (str) – Parent directory where images are located.

  • divider (str) – Divider between prefix and suffix.

  • suffixes (List[str]) – List of channel suffixes that will be loaded.

  • reload (bool) – If channels that are already loaded need to be reloaded. Defaults to False.

Creates a plate based on config options.

Warning

This immediately saves the plate.json to the disk. If overwrite_analysis is enabled, this will clear the old plate.json.

Parameters:

  • plate_name (str) – Name of the plate.

  • image_dir (str) – Directory containing the raw microscopy images.

  • output_dir (str) – Directory to save analyses and segs.

  • plate_type ("chao_lab" | "cell_painting") – Type of plate.

  • channels (List[str], optional) – Channels to be loaded for the images. This is used only for cell_painting plate formats, to map w1, w2, …, w5 to their channels.

  • overwrite_analysis (bool, optional) – If we redo the analysis and want to overwrite the previous plate data.
